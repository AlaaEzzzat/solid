Single Responsibility Principle (SRP):

A class should have only one reason to change. In other words, a class should be responsible for only one thing.

Example:

In this example : The UserListComponent is responsible for rendering the UI and handling user interactions, while the UserService is responsible for fetching and manipulating data. By separating these responsibilities, we achieve the Single Responsibility Principle, making our code more modular and easier to maintain and test.
