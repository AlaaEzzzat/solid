Open/Closed Principle (OCP):

A class should be open for extension but closed for modification. This means that you should be able to add new functionality to a class without changing its existing code.

Example:

In this example : We define a contract for a Logger interface that has a single log method. We then implement the contract in two different logger classes, ConsoleLogger and FileLogger, each with its own implementation of the log method. Next, we use the Logger interface in a ProductService without depending on a specific implementation. This makes the ProductService open for extension, since we can easily swap out the logger implementation without modifying the ProductService code. By following the OCP principle in this example, we create a more flexible and maintainable system, since we can easily add new logger implementations in the future without modifying the existing code.
