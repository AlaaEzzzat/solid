Liskov Substitution Principle (LSP):

Subtypes should be substitutable for their base types. This means that a derived class should be able to be used in place of its base class without causing any unexpected behavior.

Example:

Suppose we have an interface called Animal which has a method makeSound(). We also have two subclasses, Dog and Cat, which inherit from Animal. Both Dog and Cat implement the makeSound() method, but they do so in different ways.

Notice that both Dog and Cat inherit from Animal and implement the makeSound() method. According to the LSP, we should be able to substitute an instance of Dog or Cat wherever an instance of Animal is expected, without affecting the correctness of the program.
